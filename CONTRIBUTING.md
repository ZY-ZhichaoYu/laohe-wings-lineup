# 贡献球员卡指南

欢迎给老和山之翼阵容海报补充球员照片、绝版人物照片和球员卡介绍。不会写代码也没关系，可以把材料发给领队；会用 GitHub 的队友可以直接提 PR。

## 照片规格

推荐规格：

- 尺寸：`512 × 512` 正方形。
- 格式：`JPG`、`JPEG` 或 `PNG`。
- 文件大小：建议小于 `200KB`。
- 内容：清晰头像或半身照，脸部不要太小。
- 背景：干净即可，不强制抠图。

压缩建议：

- 手机原图通常太大，可以先裁成正方形再压缩。
- JPG 质量在 75%-85% 一般够用。
- 如果照片超过 200KB，尽量压缩后再提交，避免网页加载太慢。

## 文件命名规则

球员照片放在：

```text
assets/players/
```

绝版人物照片放在：

```text
assets/legends/
```

推荐命名：

```text
号码-姓名-home.jpg
号码-姓名-away.jpg
```

示例：

```text
10-张文杰-home.jpg
10-张文杰-away.jpg
31-黄于翀-home.png
```

如果暂时只有一张照片，可以先用：

```text
号码-姓名.jpg
```

绝版人物可以用：

```text
黄坤鱼.jpg
王开来.jpg
田治华.jpg
```

## 修改 players.json

球员资料在：

```text
data/players.json
```

找到自己的名字，修改 `avatarHome` 和 `avatarAway`：

```json
{
  "num": 10,
  "name": "张文杰",
  "avatarHome": "assets/players/10-张文杰-home.jpg",
  "avatarAway": "assets/players/10-张文杰-away.jpg"
}
```

如果只有一张照片，可以主客场先都写同一张：

```json
{
  "avatarHome": "assets/players/10-张文杰.jpg",
  "avatarAway": "assets/players/10-张文杰.jpg"
}
```

## 球员卡介绍文案

介绍文案在 `players.json` 的 `intros` 里：

```json
{
  "张文杰": "10号，中锋。曾经也打过8人制右边卫。你问张文杰和内马尔、登贝莱的区别是什么？他们的射门枪法没有张文杰准。"
}
```

风格参考：

- 轻松、幽默、有一点球队内部梗。
- 可以夸张，但不要攻击别人。
- 一句话到两句话即可。
- 尽量写出位置、特点或球场名场面。
- 不要写隐私信息、手机号、学号、宿舍等个人信息。

可以这样写：

```text
22号，替补。名字像水系法师，传球也讲究源源不断，偶尔还能制造水花。
```

不建议这样写：

```text
很强。
```

## PR 提交流程

1. Fork 这个仓库。
2. 在自己的 fork 里上传照片到 `assets/players/` 或 `assets/legends/`。
3. 修改 `data/players.json`。
4. 本地预览：

```bash
python3 -m http.server 8000
```

打开：

```text
http://127.0.0.1:8000
```

5. 确认页面能打开，球员照片能显示，随机抽卡介绍没有乱码。
6. 提交 commit。
7. 打开 Pull Request。

## PR 模板

提 PR 时可以复制下面这段：

```markdown
## 更新内容

- [ ] 新增/更新球员照片
- [ ] 新增/更新绝版人物照片
- [ ] 更新球员卡介绍
- [ ] 更新球员号码、位置、角色或状态

## 球员信息

- 姓名：
- 号码：
- 照片文件：
- 介绍文案：

## 自查

- [ ] 本地用 `python3 -m http.server 8000` 打开过页面
- [ ] `data/players.json` 没有 JSON 格式错误
- [ ] 照片路径和实际文件名一致
- [ ] 照片大小尽量小于 200KB
```

## 不会用 GitHub 怎么办

把照片、号码、姓名和介绍文案发给领队即可。领队会帮你整理文件、压缩照片、合并进仓库。
